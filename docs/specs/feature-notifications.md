# Spec: Notifications (SMS and WhatsApp)

**Epic:** WAGR-E7 Notifications
**Stories:** [sms-advance-status], [whatsapp-worker-payslip], [whatsapp-employer-summary]
**Sprint:** Week 3 ([sms-advance-status]), Week 4 ([whatsapp-worker-payslip], [whatsapp-employer-summary])
**Status:** Not started

---

## Overview

Wagr communicates with workers and employers through SMS and WhatsApp using Moolre's messaging APIs. SMS is used for transactional alerts — fast, short, and always delivered regardless of internet access. WhatsApp is used for richer content — payslips and payroll summaries that benefit from formatting.

All notification sending is asynchronous. Notification failures are logged but never block the core payment flow.

---

## User Stories

**[sms-advance-status]** — As a worker, I want to receive an SMS when my advance request is received and when it is disbursed.

**[whatsapp-worker-payslip]** — As a worker, I want to receive my payslip on WhatsApp on payday.

**[whatsapp-employer-summary]** — As an employer, I want to receive a WhatsApp payroll summary after processing payroll.

## Acceptance Criteria

### Worker SMS Notifications ([sms-advance-status])
- [ ] SMS sent on advance_request creation (status: pending)
  - Message: "Your Wagr advance request of GHS [requested_amount] has been received. You will be notified when it is sent."
- [ ] SMS sent when advance_request updated to status: disbursed
  - Message: "GHS [net_disbursed] has been sent to your MoMo. Wagr — Don't wait for payday."
- [ ] SMS sent when advance_request updated to status: failed
  - Message: "Your Wagr advance request could not be processed. Contact your employer or try again later."
- [ ] SMS delivery failure is logged to the audit_log but does not block or reverse the advance
- [ ] Phone number used is the employee's momo_number

### Worker WhatsApp Advance Summary ([whatsapp-worker-payslip])
*(Slug name retained for branch continuity. The actual artefact is an advance summary, NOT a payslip. Wagr does not disburse regular salaries; see [feature-disbursements.md](feature-disbursements.md) "Wagr's product scope".)*

- [ ] WhatsApp message sent after [payday-recovery] succeeds for the employer (one message per worker who took at least one advance this period)
- [ ] Content generated by [payslip-gpt] (LLM-backed closing line; structured fields filled deterministically from the database)
- [ ] Content includes: worker first name, pay period, employer name, total advances taken this period. **Does NOT include gross salary or net pay** — Wagr is not the payroll system and shouldn't claim authority over those figures.
- [ ] Tone is warm and plain — not a formal bank statement
- [ ] Delivered within 5 minutes of pay-period close
- [ ] Delivery failure logged — does not block or reverse anything; the advance summary is informational
- [ ] Phone number used is the employee's momo_number

### Employer WhatsApp Summary ([whatsapp-employer-summary])
- [ ] WhatsApp message sent to employer after payroll run completes
- [ ] Content includes: number of employees paid, total advances recovered, total net salaries disbursed, itemised list of employees with their net pay
- [ ] Content generated by GPT-4o with a constrained prompt
- [ ] Delivered within 5 minutes of payroll completion
- [ ] Phone number used is the employer's phone from the employers table

## Technical Notes

### Notification Service

All notification calls are centralised in a single module. No route file calls Moolre messaging APIs directly.

```typescript
// apps/api/src/lib/notifications.ts

interface SmsPayload {
  to: string           // MoMo number
  message: string
}

interface WhatsAppPayload {
  to: string           // WhatsApp number
  message: string
}

export async function sendSms(payload: SmsPayload): Promise<void> {
  try {
    await moolre.sendSms(payload)
    await auditLog.write({
      action: 'sms_sent',
      metadata: { to: payload.to, preview: payload.message.slice(0, 50) }
    })
  } catch (error) {
    await auditLog.write({
      action: 'sms_failed',
      metadata: { to: payload.to, error: error.message }
    })
    // Do not rethrow — notification failure is non-blocking
  }
}

export async function sendWhatsApp(payload: WhatsAppPayload): Promise<void> {
  try {
    await moolre.sendWhatsApp(payload)
    await auditLog.write({
      action: 'whatsapp_sent',
      metadata: { to: payload.to }
    })
  } catch (error) {
    await auditLog.write({
      action: 'whatsapp_failed',
      metadata: { to: payload.to, error: error.message }
    })
    // Do not rethrow — notification failure is non-blocking
  }
}
```

### SMS Message Templates

SMS messages are kept under 160 characters to avoid multi-part message charges.

```typescript
// apps/api/src/lib/sms-templates.ts

export const smsTemplates = {
  advanceReceived: (amount: number) =>
    `Your Wagr advance request of GHS ${amount} has been received. You will be notified when it is sent.`,

  advanceDisbursed: (netAmount: number) =>
    `GHS ${netAmount} has been sent to your MoMo. Wagr — Don't wait for payday.`,

  advanceFailed: () =>
    `Your Wagr advance request could not be processed. Contact your employer or try again later.`,

  floatFunded: (amount: number) =>
    `Your Wagr float has been funded with GHS ${amount}. Your team can now request advances.`,

  payrollComplete: (employeeCount: number) =>
    `Payroll processed for ${employeeCount} employees. Payslips sent via WhatsApp. Wagr.`,
}
```

### WhatsApp Message Triggering

WhatsApp messages are triggered from the payroll route after the payroll run completes. They are sent asynchronously — the payroll route returns a success response before the messages are sent.

```typescript
// Inside payroll completion handler
// After all disbursements confirmed:

// Fire and forget — do not await
Promise.allSettled([
  ...employees.map(emp =>
    generateAndSendPayslip(emp, payrollData)
  ),
  generateAndSendEmployerSummary(employer, payrollData)
]).then(results => {
  const failed = results.filter(r => r.status === 'rejected')
  if (failed.length > 0) {
    logger.error('Some WhatsApp messages failed', { failed })
  }
})
```
